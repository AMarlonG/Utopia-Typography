Fluid Typography

This page is a short presentation of the fluid responsive work done by Utopia. It consist of a joint design and developer goal for minimal and elegant design, which streamlines collaboration between design and developments, and ensures visual harmony and consistency. A true embrace of the ebb and flow of the internet.

Scaling type

Responsiveness

A website with no CSS is responsive by nature. This is what is referred to ass intrinsic responsiveness. The advent of different screen sizes made fitting content for these differences a high priority. This is what's called responsive web design. This has for a long time, and still does, meant defining breakpoints and the use of media queries in CSS code.

I'm a text and I use Utopia calculations to fluidly transition between screen sizes

I'm also a text, but I don't use any responsive coding in CSS. I'm what's called intrinsic.

I'm the most common text, and I realy on different media queries to define breakpoints.

The main differences is that (1) the Utopia text fluidly transitions between font sizes, while (2) the intrinsic never changes, and (3) the media queries have distinct hard changes.

Under the hood

The Utopia principles will hopefully simplplify the work flow for designers and developers. Designers don't have to design for several breakpoints, and can trust the computer to automatically calculate "the feel" of the design regardless of screen sizes. For developers this process will in many instances mean just one line of code, font-size: var(--step-1) after using the calculations available on the Utopia website. Minimal and elegant code, with more time to collaborate with designers.

All the complicated maths behind the Utopia principles can be found on the Utopia website.
